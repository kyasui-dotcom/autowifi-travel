# 観光メディア方針と公開スケジュール

## 目的

- 観光系の検索流入を増やし、自然に `AutoWiFi eSIM` へ送客する
- `eSIMを売るための記事` ではなく、`旅行者が普通に読みたくなる観光記事` を作る
- X はネタ発見と空気感の補助に使い、記事の主役は独自編集にする
- 一般的な観光ガイドに載りやすい定番ではなく、`ややマイナーな場所 / 切り口 / 回り方` を主戦場にする
- 日本限定ではなく、`外国人旅行者向けに成立するグローバルなマイナートラベル記事` を主軸にする

## 編集方針

- 記事テーマは `1記事1テーマ1導線`
- 主役は `マイナーな観光地 / 静かな散歩ルート / 季節特集 / 半日モデルコース / ニッチなエリア紹介`
- X は `公式埋め込み 3件以上`
- 写真は `10枚以上`
- eSIM 導線は `記事末` と `関連記事` に薄く置く
- 本文は `自社の整理・解説・行き方・注意点` を中心にする
- `行きたい理由` が明確な記事を優先する
- `バズったから書く` のではなく、`旅行者が行動に移せる形に編集できるネタ` だけ採用する
- `大手観光ガイドに普通に載る王道スポット` は避け、`その都市に数回行く人でも知らないかもしれない場所` を優先する
- 採用基準は `少しマイナーだけど、現地でちゃんと行ける` こと
- `3,000文字 / 写真10枚 / X3件` を満たせないテーマは採用しない

## ネタ選定ルール

- `有名観光地そのもの` より `その周辺の静かなエリア` を選ぶ
- `駅から歩ける / 半日で回れる / 朝や夕方に映える` など、旅行者が動きやすい条件を優先する
- `Xで雰囲気が伝わる` ことを重視する
- `地図 / 営業情報 / 行き方` の裏取りができないものは採用しない
- `ローカル感はあるが危険ではない` こと
- `インバウンド向けにまだ飽和していない` 切り口を優先する

## 記事フォーマット

1. 導入
2. なぜ今この場所がよいか
3. どんな人に向くか
4. 行き方
5. ベスト時間帯
6. 注意点
7. 周辺で合わせて寄れる場所
8. 写真
9. X の公式埋め込み
10. FAQ
11. 関連記事
12. さりげない旅行準備導線

## X 運用ルール

- 無断転載しない
- スクショ転載しない
- 画像を抜き出して再掲載しない
- 公式埋め込みを使う
- 1記事につき `3件以上` を基本にする
- 投稿が消えても記事が成立するように本文を作る
- 公開前に `公式サイト / Google Maps / 観光協会 / 店舗情報` のいずれかで裏取りする
- 記事ごとに内容と一致する投稿を拾い、`同じ汎用X投稿を全記事へ使い回さない`

## 優先カテゴリ

### Global clusters

- quiet neighborhoods outside the usual first-timer list
- morning walks in low-key districts
- station-based half-day routes with 2 to 3 stops
- rainy day calm spots with cafes, bookstores, or galleries
- coffee and gallery areas that still feel local
- old-town streets and tram-adjacent areas
- ferry and island day routes
- hillside towns and outer-neighborhood walks
- local transit-led half days
- market streets that stay practical for foreign travelers
- neighborhood themes that can scale across Asia, Europe, and the Americas

## 公開基準

- 最低 `3,000文字` を下限にする
- 写真 `10枚以上`
- X 埋め込み `3件以上`
- FAQ `5件以上`
- 関連記事 `3本以上`
- 記事末に旅行準備導線を置く

## 運用ペース

- `1日5記事` を基本にする
- 公開時間は固定しなくてよい
- 重要なのは `毎日5本出ること` と `テーマが被りすぎないこと`
- `100本バックログ` を先に作り、automation はそこから未公開スラッグを選ぶ
- まずは `20日で100本` を一区切りにする
- 1日の構成は `主力1本 + 派生2本 + 比較/回遊2本` を基本にする

## 1日の基本構成

1. 主力記事 `1本`
2. 同クラスターの派生記事 `2本`
3. 回遊を増やす比較・まとめ記事 `2本`

### 例

- Quiet Tokyo neighborhoods
- Yanaka half-day route
- Kuramae stationery walk
- Best calm Tokyo morning walks
- Rainy day Tokyo areas worth walking

## 8週間スケジュール

### Day 1 to Day 20

- 毎日 `5本`
- 合計 `100本`
- 地域は `日本 / 韓国 / 台湾 / 香港 / 東南アジア / 南ヨーロッパ / ラテンアメリカ` を混ぜる
- 1日ごとに `同一都市の派生3本 + 関連都市2本` を基本にする
- 既存公開記事と slug 重複があるものは飛ばす

## バックログ

- `docs/minor-travel-guide-backlog.md` を source of truth にする
- automation は未公開 slug を優先して使う
- 品質条件を満たせないテーマはスキップする

## KPI

- 20日で観光記事 `100本`
- 1記事あたり `3,000文字以上`
- 主要記事に `画像 + FAQ + 関連記事 + X埋め込み` を標準装備
- 観光記事から eSIM ページへの内部リンククリックを計測対象にする

## やらないこと

- Xまとめサイト化
- 300〜500語の薄い観光記事量産
- 写真なしの記事量産
- eSIM を本文中で強く売り込む構成
- 裏取りなし公開
